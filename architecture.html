JWT Token Validation Flow
Flutter App
POST /auth/signin
1. Credentials
→
Auth Lambda
AWS Cognito SDK
2. Authenticate
→
JWT Tokens
accessToken · idToken · refreshToken
3. Response
→
FlutterSecureStorage
auth_token · user_id
4. Stored securely
→
Protected Request
Authorization: Bearer <token>
5. API calls
→
Cognito Authorizer
Validates JWT signature + expiry
6. Validated at Gateway
ℹ️ The Gamification API uses a different approach: the app decodes the JWT locally (GameService.extractSubFromToken) to extract the Cognito sub claim and passes it as cognitoId in the request body — no Authorization header required for score submission. All LMS and Forum endpoints use Authorization: Bearer headers validated by the API Gateway Cognito Authorizer.
